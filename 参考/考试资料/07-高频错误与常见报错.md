# 07 高频错误与常见报错

## 24. 考试高频错误与检查清单

### 高频错误

- 表单 `name` 和后端 `@RequestParam` 不一致
- 返回模板名写错
- 模板文件不在 `templates`
- 静态文件和模板文件放错目录
- 忘记写 `@Entity`
- 忘记写 `@Id`
- 关系表忘记加唯一约束，导致同一个用户对同一部电影出现重复记录
- Repository 没继承 `JpaRepository`
- 用了 `starter-security` 时，POST 表单忘记 CSRF
- 只用了 `spring-security-crypto` 却误以为 Spring 会自动帮你拦截登录
- 登录页、注册页、静态资源忘记 `permitAll()`
- 密码比较用了 `equals`
- `UserDetailsService` 里返回了明文密码，不是数据库里的 hash
- `Principal` 在未登录时直接使用
- `TP4` 明明要求 `HttpSession`，却把模板写成了 `Principal` / `SecurityFilterChain` 路线
- `redirect:` 和直接返回页面混了
- API key 没配
- `@Value` 对应的配置 key 没写
- DTO 字段名和 JSON 对不上
- 收藏和 watchlist 的 POST 路由写了，但页面表单 action 没对上
- `movie-detail.html` 里按钮提交成功了，但忘记重定向回详情页或列表页

### 提交前检查

1. 项目能不能启动
2. 首页能不能打开
3. 表单能不能提交
4. 页面能不能显示数据库数据
5. 新增后数据库里有没有记录
6. 登录能不能成功
7. 密码是不是加密保存
8. API 能不能调通
9. DTO 能不能正确接收数据
10. 每个 POST 表单有没有处理
11. 登录后能不能进入受保护页面
12. 收藏、取消收藏、watchlist 至少有一条完整链路能跑通
13. `stored` 页面或用户空间页面能不能读出数据库里的数据
14. 如果是 `TP4` 路线，session 里有没有成功保存当前用户 `id`

## 25. 常见报错

### Whitelabel Error Page

- 一般原因：路由不存在、模板不存在、Controller 报异常
- 处理：先看控制台真实异常

### 404 Not Found

- 一般原因：URL 写错、没有对应映射

### 405 Method Not Allowed

- 一般原因：表单发 `POST`，后端只写了 `@GetMapping`
- 也可能是浏览器访问了一个只支持 `POST` 的地址

### Template not found

- 一般原因：返回了模板名，但 `templates` 里没有同名文件
- 也可能是返回了 `users/login`，但实际文件名是 `login.html`

### Port already in use / Web server failed to start

- 一般原因：端口被别的进程占用了
- 处理：改 `server.port`，或关闭之前已经启动的项目

### Failed to configure a DataSource

- 一般原因：没写数据源配置、缺 JDBC 依赖

### Cannot determine Dialect

```properties
spring.jpa.database-platform=org.hibernate.community.dialect.SQLiteDialect
```

### No suitable driver

- 一般原因：没加 `sqlite-jdbc`

### Unable to open database file

- 一般原因：SQLite 文件路径写错，或者 `db/` 目录不存在
- 也可能是数据库文件被别的程序占用

### database is locked

- 一般原因：SQLite 文件被重复写入，或者之前的程序没正常释放连接
- 处理：先停掉重复启动的项目，再重试

### Table not found / Column not found

- 一般原因：Entity 和表结构不一致
- 也可能是改了实体字段，但旧数据库文件还保留着旧结构

### `BeanCreationException`

- 一般原因：某个 Bean 注入失败、配置类写错、依赖缺失
- 处理：往下看 `Caused by`，真正原因通常在后面

### `UnsatisfiedDependencyException`

- 一般原因：构造器注入的对象没注册成 Bean
- 常见情况：漏写 `@Service`、`@Repository`、`@Configuration`

### `No qualifying bean of type ...`

- 一般原因：Spring 容器里没有这个类型的 Bean
- 常见情况：`PasswordEncoder` 没有 `@Bean`，`RestTemplate` 没有配置

### `UsernameNotFoundException`

- 一般原因：登录用户查不到

### `Bad credentials`

- 一般原因：用户存在，但密码匹配失败
- 也可能是数据库保存的是明文，但 Security 按 hash 在校验

### `NullPointerException`

- 一般原因：取到的是 `null` 但直接调用方法
- 常见情况：`findByEmail(...)` 没查到用户，`session.getAttribute(...)` 没取到值

### `Optional` 取值异常

- 一般原因：`optional.get()` 时没有值

### `Could not resolve placeholder`

- 一般原因：配置文件没写对应 key

### JSON 映射失败

- 一般原因：DTO 字段名和 JSON 对不上
- 也可能是 JSON 结构里嵌套了一层，但 DTO 少写了一层对象

### `HttpMessageConversionException`

- 一般原因：返回的 JSON 和 DTO 类型对不上
- 常见情况：接口给的是数组，你却用单个对象接收

### `ConstraintViolationException` / 唯一约束冲突

- 一般原因：唯一字段重复，例如 `email` 重复，或关系表里同一 `(member_id, movie_id)` 重复

### `DataIntegrityViolationException`

- 一般原因：非空字段传了 `null`，或外键约束失败
- 常见情况：关系实体里忘了设置 `appMember` 或 `storedMovie`

### `PropertyReferenceException`

- 一般原因：Repository 方法名写错，Spring Data 无法解析
- 常见情况：实体里叫 `email`，方法却写成 `findByMail`

### `LazyInitializationException`

- 一般原因：懒加载关联在 Session 外被访问
- 考试里如果遇到，先检查是不是在模板里直接读了还没初始化的关联字段

### 登录后又跳回登录页

- 一般原因：路由被保护了但登录流程没配好，或者登录页参数名和 `usernameParameter()` 不一致

### Session 里取不到当前用户

- 一般原因：登录成功后没有 `session.setAttribute(...)`
- 或者存进去的是一个 key，读取时用了另一个 key

### API 调用失败

- 一般原因：API key 错、URL 错、网络问题
- 也可能是查询参数拼错、语言参数写错、电影 `id` 不存在

### 远程 API 返回 401 / 403

- 一般原因：API key 无效、没传 key、权限不够

### 远程 API 返回 404

- 一般原因：URL 路径写错，或者请求的资源不存在

### 远程 API 返回 429

- 一般原因：请求太频繁，被限流

### 页面能打开但数据不显示

- 一般原因：`model.addAttribute(...)` 名字和模板里 `${...}` 不一致
- 也可能是 Controller 根本没把数据放进 Model

### 表单提交后没有任何变化

- 一般原因：Controller 没有接到对应参数，或者保存后没有重定向到正确页面
- 检查：表单 `action`、字段 `name`、Controller 参数名、`save(...)` 有没有真的执行

### 更新功能把旧密码清空了

- 一般原因：更新用户时表单没填密码，但代码直接把空字符串保存回数据库
- 处理：更新前先判断密码是否为空；为空时保留原值

### 二次加密导致无法登录

- 一般原因：数据库里已经是 bcrypt hash，又被 `encode(...)` 了一次
- 常见情况：`updateUser(...)` 时没有判断当前值是不是 hash

### 收藏按钮点一次报错，点两次重复

- 一般原因：关系表缺唯一约束，或者保存前没先查已有关系

### 页面出现白页但控制台没明显报错

- 一般原因：模板表达式写错、返回视图名和模板文件不一致
- 处理：先看浏览器请求地址，再看 Controller 返回值，再对模板文件名

## 26. 出错时先看哪里

1. 先看控制台最底部的 `Caused by`
2. 再看是启动时报错，还是点页面之后才报错
3. 启动时报错优先查：依赖、配置、Bean、数据库连接
4. 页面点击后报错优先查：Controller、Service、Repository、模板字段名
5. 登录问题优先查：密码是否 hash、`matches(...)`、session 是否写入
6. 数据不显示优先查：`model.addAttribute(...)` 和 Thymeleaf 变量名是否一致
